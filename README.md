# Tech Challenge 3 — Terraform + Ansible (Hello, World!)

This project provisions an Ubuntu EC2 instance and an S3 bucket with **Terraform**, then configures the instance with **Ansible** to run **Nginx** serving a simple “Hello, World!” page.

## Repo Layout

```
tech-challenge-3/
├─ terraform/                 # IaC for AWS
│  ├─ versions.tf
│  ├─ variables.tf
│  ├─ main.tf
│  ├─ outputs.tf
│  └─ terraform.tfvars.example
├─ ansible/                   # Config mgmt for EC2
│  ├─ ansible.cfg
│  ├─ inventory.ini           # generated by script; edit manually if needed
│  └─ site.yml
├─ scripts/
│  └─ write_inventory.sh      # helper to generate Ansible inventory from TF outputs
├─ docs/
│  └─ TEARDOWN.md
└─ .gitignore
```

## Prerequisites

- **AWS account** with programmatic access (Access Key ID/Secret) for a user that can create EC2, S3, IAM, and security groups.
- **AWS CLI**: `aws configure` to set your credentials and default region.
- **Terraform >= 1.6**
- **Ansible >= 2.14**
- **SSH key pair** on your machine.

Mac (Homebrew):

```bash
brew install awscli terraform ansible
```

Linux (Ubuntu/Debian):

```bash
sudo apt update && sudo apt install -y awscli terraform ansible
```

Windows:

- Use **WSL2** (Ubuntu) and follow the Linux instructions above.

## Step-by-Step

### 1) Configure AWS Credentials

```bash
aws configure
# enter Access Key ID, Secret, default region (e.g., us-east-1), and output format (json)
```

### 2) Generate an SSH Key (if you don’t have one)

```bash
ssh-keygen -t ed25519 -f ~/.ssh/tc3 -C "tc3"
# creates ~/.ssh/tc3 (private) and ~/.ssh/tc3.pub (public)
```

### 3) Set Terraform Variables

Copy the example to real vars and edit the values (use ABSOLUTE path to your PUBLIC key):

```bash
cd terraform
cp terraform.tfvars.example terraform.tfvars
# edit terraform.tfvars and set: public_key_path = "/ABSOLUTE/PATH/TO/your_public_key.pub"
```

### 4) Deploy Infrastructure

```bash
terraform init
terraform plan
terraform apply -auto-approve
```

**Outputs** will show the EC2 public IP and a convenience URL.

```
terraform -chdir=terraform output -raw public_ip
terraform -chdir=terraform output -raw web_url
terraform -chdir=terraform output -raw bucket_name
```

Example:

```
public_ip = "3.91.x.y"
web_url   = "http://3.91.x.y/"
bucket_name = "tc3-ab12cd"
```

### 5) Prepare Ansible Inventory

From the **project root** (not inside terraform/), run:

```bash
./scripts/write_inventory.sh ~/.ssh/tc3
```

This writes `ansible/inventory.ini` with the correct EC2 IP and your private key path.

> If you don’t use the script, edit `ansible/inventory.ini` manually and replace:
> - `EC2_PUBLIC_IP` with your instance’s `public_ip`
> - `PRIVATE_KEY_PATH` with your private key (e.g., `~/.ssh/tc3`)
> - user is `ubuntu` for Ubuntu AMIs

### 6) Test Connectivity

```bash
cd ansible
ansible -m ping web
# Expect: web | SUCCESS => {"changed": false, "ping": "pong"}
```

### 7) Configure Web Server

```bash
ansible-playbook site.yml
```

This installs Nginx, deploys an `index.html`, and ensures the service is running.

### 8) Validate

- Open the `web_url` from Terraform outputs in your browser.
- Or curl it:
  ```bash
  curl -I http://$(terraform -chdir=terraform output -raw public_ip)/
  ```

You should see the “Hello, World!” page.

## Teardown (Stop AWS charges)

See [`docs/TEARDOWN.md`](docs/TEARDOWN.md). Quick command:

```bash
terraform -chdir=terraform destroy -auto-approve
```

> The S3 bucket is created with `force_destroy = true`, so Terraform deletes objects and the bucket automatically.

## Notes

- Security group allows SSH (22) from `allow_ssh_cidr` and HTTP (80) from `allow_http_cidr`.
- For testing, allow_ssh_cidr may be 0.0.0.0/0.
  - ⚠️ Best practice: restrict to your own IP before sharing:

  ```
  curl ifconfig.me   # get your public IP
  # update terraform.tfvars
  allow_ssh_cidr = "<YOUR_IP>/32"
  ```

- EC2 gets an Instance Profile with S3 read/list permissions for the created bucket (useful if you later want to pull content).
- Uses default VPC and first subnet — fine for this single-instance demo.

## Troubleshooting

- Terraform error: no matching VPC found
  → Run aws ec2 create-default-vpc --region us-east-1 to create one.
- SSH/Ansible ping fails
  - Ensure Security Group allows your IP (allow_ssh_cidr)
  - Confirm private key permissions: chmod 600 ~/.ssh/tc3
  - Ensure correct user (ubuntu for Ubuntu AMIs)
- Nginx not serving page
  - SSH in: ssh -i ~/.ssh/tc3 ubuntu@<PUBLIC_IP>
  - Check service: sudo systemctl status nginx
  - Check page: sudo cat /var/www/html/index.html
